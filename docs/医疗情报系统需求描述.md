# 技术评估：科研情报智能体

## 背景
公司临床中心正在规划一个科研情报智能体系统，核心目标是帮助早期科学部、医学部的研究人员快速获取靶点相关的竞品情报——包括在研临床试验、近期文献进展、竞争格局分析等——以辅助早期开发的立项决策。

目前这些工作完全依赖人工：研究人员需要逐个登录 ClinicalTrials.gov、PubMed 等数据库手动检索，再花大量时间整理成报告。我们希望用 AI 将这个过程自动化。

请你基于以下需求，完成一个可工作的原型系统。

## 功能需求

### 核心场景
用户输入一个靶点名称（如 HER2、PD-L1、KRAS G12C），系统自动完成以下工作并输出一份结构化的竞品情报报告。

### F1. 多源数据采集
系统需从至少两个及以上公开数据源自动、全面采集信息。建议涵盖但不限于以下数据源及信息维度（也可根据实际需求增加相关医药数据源，内容越丰富全面越佳）：

- **ClinicalTrials.gov** —— 在研临床试验信息（应包括试验标题、申办方、研究阶段、状态、入组人数、试验干预、国家地区、注册号等完整字段）
- **PubMed** —— 相关学术文献（建议采集标题、作者、期刊、发表日期、摘要、关键词、引用次数等尽可能多的元数据）

**可选拓展数据源举例（非必须、鼓励加分）：**
- FDA/EMA/NMPA 公告 —— 药品批准、上市及警告信息
- 专利数据库 —— 相关靶点的专利信息（专利号、技术方案、申请人等）
- 医学/药学会议摘要 —— ASCO、ESMO、AACR 等会议的最新研究进展
- 药品注册与管线库 —— 如药渡（Pharmcube）、CDE、Informa 等第三方数据库

**要求：**
- 自动采集、整合多个来源的信息，保证数据广度与覆盖面；
- 每个数据源采集内容尽量详实，结构化字段齐全，便于后续分析与报告生成；
- 具备后续无侵入式拓展更多新数据源的可扩展性。

> 注：如可获取更多高价值、前沿、全面的数据源并集成，将获得更高评分。

### F2. 智能报告生成
利用大语言模型对采集到的数据进行分析与归纳，生成一份结构化报告。报告至少应包含：

- **靶点概述**：靶点的基本信息与作用机制
- **在研管线概览**：按研发阶段分布的在研项目统计，主要竞争企业及其产品
- **近期研究动态**：近期关键文献发现的摘要与解读
- **竞争格局判断**：当前竞争态势、潜在风险与机会

报告需以可阅读的格式输出（如 Markdown、HTML、PDF 等）。

### F3. 交互方式
提供用户可操作的交互入口。可以是命令行工具、Web 界面、API 服务或其他你认为合适的形式。

### F4. 可扩展的数据源架构
系统在设计上应考虑后续接入更多数据源（如 ASCO/ESMO 会议摘要、药物专利库、FDA 审批信息等）。希望新增一个数据源时，不需要大规模改动现有代码。

### F5. 基本的工程化要求
- 合理的错误处理（网络异常、API 不可用、数据为空等场景）
- API Key 等敏感信息不硬编码在源码中
- 有基本的日志输出，方便排查问题

## 进阶需求（非必须，但鼓励挑战）
以下需求不做硬性要求，但如果你有时间和想法，欢迎实现：

1. **缓存与增量更新** —— 对已查询过的靶点结果进行缓存；支持增量检测，仅处理新增数据
2. **多数据可视化** —— 在报告中加入图表（如管线阶段分布图、年度文献趋势图等）
3. **报告版本管理** —— 同一靶点的历史报告可追溯对比
4. **异步处理与任务管理** —— 报告生成为异步任务，支持状态查询
5. **自动化测试** —— 对核心模块有测试覆盖

## 设计文档
除可运行代码外，请提交一份架构设计文档，说明：

1. 系统整体架构与各模块职责
2. 关键技术选型及理由
3. 数据从采集到报告输出的完整流转过程
4. 当检索结果数据量很大时，你如何处理（如超出 LLM 上下文窗口）
5. 你预见的主要技术风险及应对策略
